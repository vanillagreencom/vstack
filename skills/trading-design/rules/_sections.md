# Sections

This file defines all sections, their ordering, impact levels, and descriptions.
The section ID (in parentheses) is the filename prefix used to group rules.

---

## 1. Design Philosophy (phil)

**Impact:** CRITICAL
**Description:** The foundational "why" behind professional trading UI — density-first thinking, signal-through-noise principles, and the identity that separates professional tools from retail platforms.

## 2. Visual Language (vis)

**Impact:** CRITICAL
**Description:** Color theory, opacity system, near-black foundation, and surface elevation — the visual principles that create hierarchy without introducing noise.

## 3. Typography & Density (type)

**Impact:** CRITICAL
**Description:** Dual-font system, data alignment, size hierarchy, and semantic token architecture — the typographic and structural patterns that enable extreme data density.

## 4. Layout & Panels (layout)

**Impact:** HIGH
**Description:** Panel architecture, docking/tiling, priority-based space allocation, required states, and responsive collapse strategies.

## 5. Data Display (data)

**Impact:** HIGH
**Description:** Conventions for rendering prices, positions, orders, P&L, alerts, and stale data — the patterns professional traders expect.

## 6. Interaction Design (ix)

**Impact:** HIGH
**Description:** Keyboard-first workflows, error prevention, confirmation flows, focus management — interaction patterns that prioritize speed and safety.

## 7. Component Philosophy (comp)

**Impact:** MEDIUM
**Description:** When to compose vs custom-render, ShadCN-density adaptation, trading widget patterns, and component completion criteria.

## 8. Accessibility (a11y)

**Impact:** MEDIUM
**Description:** Contrast requirements in dark environments, never-color-alone principle, focus indicators, and cross-platform rendering considerations.
